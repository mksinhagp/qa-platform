# Task 15: E2E Smoke Tests (Playwright)

## Objective
Create Playwright E2E smoke tests for dashboard-web covering:
- Bootstrap vault
- Login  
- Unlock vault
- Create saved secret (credential)
- Lock vault
- Attempted reveal denied while locked

## Test Environment
- Dashboard web runs on http://localhost:3000
- PostgreSQL on port 5434
- Need docker compose up for full stack

## Required Setup
1. Install Playwright: `npm init playwright@latest apps/dashboard-web`
2. Configure playwright.config.ts for http://localhost:3000
3. Use test database or clean state between tests

## Test File Location
apps/dashboard-web/e2e/smoke.spec.ts

## Test Flow (single test or multiple sequential)
```
1. Navigate to /dashboard/settings/vault/bootstrap
2. Bootstrap vault with master password
3. Navigate to /login
4. Login with operator credentials
5. Navigate to /unlock
6. Unlock vault with master password
7. Navigate to /dashboard/settings/credentials/new
8. Create a saved credential
9. Lock vault (via vault state pill or /unlock)
10. Attempt to reveal credential
11. Verify access denied / redirected to unlock
```

## Key Requirements
- Use Playwright fixtures and page objects
- Use test-data-id attributes for reliable selectors
- Tests should be independent and clean up after
- Use beforeAll/beforeEach for setup
- Configure retries for flaky tests
- Test in headed mode for debugging, headless for CI

## Exit Criteria
- Playwright installed and configured
- smoke.spec.ts created with full test flow
- Tests pass when running `npx playwright test`

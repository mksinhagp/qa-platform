# Task 14: Integration Tests (Vitest)

## Objective
Create comprehensive Vitest integration tests for Phase 1 components per Master Plan §17.2:
- Argon2id round-trip and cost calibration
- Vault bootstrap → unlock → encrypt secret → decrypt secret → lock → access denied
- KDF parameter rotation
- Master password rotation re-wraps RVK only
- Session timeout behavior
- Capability checks enforced server-side
- Audit row written for every security-sensitive action

## Test Infrastructure
- Vitest is already configured in vitest.config.ts
- Test setup in vitest.setup.ts with test env vars
- Existing tests: packages/auth/src/password.test.ts, packages/vault/src/crypto.test.ts
- Use these as patterns for new tests

## Required Test Files to Create

1. **packages/auth/src/sessions.test.ts**
   - Test createSession, validateSession, revokeSession
   - Test session timeout behavior (idle and absolute)
   - Use sp_operator_sessions_* stored procedures

2. **packages/auth/src/capabilities.test.ts**
   - Test getCapabilitiesForOperator
   - Test hasCapability, hasAnyCapability, hasAllCapabilities
   - Test role assignment and capability resolution

3. **packages/vault/src/vault.test.ts**
   - Test full vault lifecycle: bootstrapVault → unlockVault → encryptSecret → decryptSecret → lockVault
   - Test access denied when vault locked
   - Test KDF parameter rotation
   - Test master password rotation (re-wraps RVK only, secrets unchanged)

4. **packages/db/src/integration.test.ts**
   - Test stored procedure invocation wrappers
   - Test transaction helper
   - Test connection pooling

5. **apps/dashboard-web/app/actions/audit.test.ts**
   - Test audit log insertion
   - Test audit log querying with filters
   - Verify audit rows written for security-sensitive actions

## Key Requirements
- Use test environment variables from vitest.setup.ts
- Mock database calls where appropriate using vi.mock()
- Test both success and failure paths
- Ensure tests are isolated (reset state between tests)
- Use describe/it blocks with clear naming
- Test timeout: 10000ms (configured in vitest.config.ts)

## Exit Criteria
All test files created and tests pass when running `pnpm test`
